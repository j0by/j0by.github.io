# What to blog?

2015-01-07

So, I moved my blog from Google - Blogger, to Github Pages. It is powered by Jekyll. But now the problem is I don't really have anything to blog about. I have so far written about how Finance and Economics cons people, but the audience I am reaching out to is not really interested in that. So I thought I will write about how to get your blog up and running on Github Pages.

Why would you want to use Jekyll for your blog? Well, there is no real reason to switch, except that it is simple and elegant. You don't have worry about logins, databases and comments. So, if you feel you can move beyond Blogger and Wordpress, go ahead and try it. To do this, you don't really need to know programming. If you follow the steps below you should be good.

Before we start, it is good to know a few things. We will be writing the actual posts in Markdown. For those who don't know much programming, that is a good thing because you don't really have to learn HTML. You can learn the basics of Markdown here. The tutorial will take you about 10 mins and will teach you enough to create your posts in markdown. You can also refer to this cheatsheet. Jekyll is a static site generator. It will help us create the blog/site. It's written in the Ruby programming language. Finally, we will be hosting the blog on Github Pages.
Let's begin.

  1. First, you need to ensure that you have Ruby and Git installed on your system. If you are on a Mac, you will mostly have Ruby pre-installed. You can type ruby --version on the Terminal to see if Ruby is installed. If not you can download it (Windows/Mac). After that install git. You can download it here.
  2. Now you need to install Jekyll. If you are planning to host your site on GitHub pages you should install the GitHub-pages gem. Type gem install gitHub-pages from the terminal. This will also install Jekyll.
  3. To create the files for your blog, from the terminal do Jekyll new your_blog_name. But this will use the standard theme. If you want to use a different theme, then download a Jekyll theme. Unzip and rename the folder to blog name. Now edit the _config.yml file to change the title, email and other contact details to the one's you want. All your posts will in the _posts folder. You will have to name your posts using the following convention yyyy-mm-dd-title.markdown. Like I mentioned before you will be writing your posts in markdown. There will be sample posts in the _posts folder. You can duplicate them to create your posts. Be sure to follow the naming conventions. Also, once you open the file, edit the title and date, to the correct values. Also, edit the about.md file. Now type the following from the terminal.
jekyll your_blog_name  
jekyll serve

This will start a local server. Open a browser window and type in http://localhost:4000 to view your site.
4. To host your blog on GitHub pages, you first need to sign up on Github. Choose the name of your blog as your username. Say if your blog is called Awesome blog (ya not very imaginative), then choose awesomeblog as your username. Once you sign in/sign up, there will be a tab called Repositories. Click on it. Then click on new. In the name of your repository enter username.github.io where username is your GitHub username. For hosting, you need to give this name to your repository. Now open up the terminal and type in the following.

$ cd your_blog_name     
$ git init 
$ git add . 
$ git commit -m 'First commit'  
$ git remote add origin https://github.com/username/username.github.io.git  
$ git push -u origin master

Now go to https://username.github.io and you can see your site. This is the URL where your site will be. To make the changes, you have made to your folder (like adding a post) reflect on your blog do

$ git add .  
$ git commit -m 'Type a message for your commit'  
$ git push

Basically, that's all you need to know to start blogging using Jekyll and GitHub pages.
